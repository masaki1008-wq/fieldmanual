# FieldManual — 現場動画マニュアルシステム

スマートフォン・PC 両対応の現場向け動画マニュアル作成アプリです。  
カメラを使用するため **HTTPS接続が必要**です。このパッケージにはローカル HTTPS サーバーが含まれています。

---

## 📁 ファイル構成

```
fieldmanual/
├── index.html   ← アプリ本体
├── server.js    ← Node.js HTTPS サーバー
├── cert.pem     ← SSL証明書（自動生成）
├── key.pem      ← SSL秘密鍵（自動生成）
├── start.sh     ← 起動スクリプト (Mac / Linux)
└── start.bat    ← 起動スクリプト (Windows)
```

---

## 🚀 起動手順

### 必要なもの

- **Node.js** v16以上 → https://nodejs.org
- **openssl**（証明書生成に必要。Mac/Linux は標準搭載）

---

### Mac / Linux の場合

ターミナルを開いて以下を実行:

```bash
# このフォルダに移動
cd /path/to/fieldmanual

# 実行権限を付与（初回のみ）
chmod +x start.sh

# 起動
./start.sh
```

---

### Windows の場合

`start.bat` をダブルクリック、またはコマンドプロンプトで:

```cmd
cd C:\path\to\fieldmanual
start.bat
```

---

### 手動起動（共通）

```bash
node server.js
# または別ポートを指定する場合
node server.js 8444
```

---

## 📱 スマートフォンからのアクセス方法

サーバー起動後、ターミナルに以下のようなアドレスが表示されます:

```
========================================
  FieldManual — HTTPS サーバー起動
========================================

  ✅ PC からアクセス:
     https://localhost:8443

  📱 スマートフォンからアクセス (同一Wi-Fi):
     https://192.168.1.10:8443
========================================
```

**スマートフォンとPCを同じ Wi-Fi に接続した状態**で、  
スマートフォンのブラウザに `https://192.168.x.x:8443` と入力してください。

---

## ⚠️ 「安全でない接続」警告について

自己署名証明書を使用しているため、初回アクセス時にブラウザが警告を表示します。  
これは社内 LAN 限定の利用であれば問題ありません。

### Safari (iPhone) の場合
1. 「このWebサイトを開く」または「詳細を表示」をタップ
2. 「Webサイトに移動」をタップ

### Chrome (Android) の場合
1. 「詳細設定」をタップ
2. 「〇〇にアクセスする（安全ではありません）」をタップ

### PC Chrome の場合
1. 「詳細設定」をクリック
2. 「localhost にアクセスする（安全ではありません）」をクリック

---

## 🔒 カメラ権限について

アプリ内で録画ボタンを押すと、ブラウザがカメラとマイクの使用許可を求めます。  
**「許可」** を選択してください。

一度拒否した場合は、ブラウザのアドレスバー横の 🔒 アイコンから権限を再設定できます。

---

## 🛑 停止方法

ターミナルで `Ctrl + C` を押してください。

---

## 🌐 本番環境での利用（社内サーバーへの展開）

より本格的な運用には、社内ネットワーク上のサーバーに展開することを推奨します。

```bash
# 社内サーバー（Ubuntu / CentOS）での起動例
node server.js 443
```

または、**Let's Encrypt** や社内 CA の証明書に差し替えることで、  
警告なしで HTTPS 接続できるようになります。

```bash
# 証明書ファイルを差し替える場合
# cert.pem → 発行された証明書
# key.pem  → 対応する秘密鍵
```

## ❓ よくある問題

| 症状 | 原因 | 対処 |
|------|------|------|
| カメラが起動しない | HTTP でアクセスしている | `https://` で接続しているか確認 |
| ポートが使用中 | 別のプロセスが占有 | `node server.js 8444` で別ポートを指定 |
| スマホからつながらない | 別のネットワーク | PC とスマホが同じ Wi-Fi か確認 |
| 権限エラー（Mac/Linux） | ポート番号が 1024 未満 | 8443 など 1024 以上のポートを使用 |
